# TabTidy

TabTidy는 Chrome 탭 그룹을 대체하지 않는다. Chrome 전체 탭 작업공간을 분석하고, 사용자가 안전하게 정리할 수 있게 돕는 확장프로그램이다.

Manifest V3 기반으로 동작하며, 빌드 과정 없이 `chrome://extensions`에서 프로젝트 루트 폴더를 `Load unpacked`로 바로 불러올 수 있다.

## 문제 정의

Chrome 탭 그룹은 탭을 묶는 데 좋지만, 이미 열려 있는 전체 작업공간의 상태를 점검해 주지는 않는다. 시간이 지나면 중복 탭, 오래 안 본 탭, 검색 결과 탭, 그룹 밖에 흩어진 탭이 쌓인다.

TabTidy는 이런 상태를 분석하고, 사용자가 직접 확인한 탭만 정리하도록 돕는다.

## Chrome 탭 그룹과의 차이

- Chrome 탭 그룹을 만들거나 저장하는 기능을 대체하지 않는다.
- 기존 Chrome 탭 그룹 위에서 전체 탭 작업공간을 점검한다.
- 자동으로 탭을 닫거나 이동하지 않는다.
- 정리는 항상 `선택 → 미리보기 → confirm → 실행` 흐름을 따른다.

## 핵심 기능

- 현재 창 전체 탭 수, 그룹 탭 수, 비그룹 탭 수, 탭 그룹 수 표시
- Chrome 탭 그룹별 이름, 색상, 탭 수, 건강 점수 표시
- 비그룹 탭 목록과 도메인별 묶음 표시
- 동일 URL 및 추적 파라미터 제거 후 중복 탭 감지
- Google, Naver, Bing, YouTube, GitHub 검색 결과 탭 감지
- 오래 안 본 탭 추적 및 3일/7일 기준 표시
- 보호 탭 토글
- 탭별 메모 저장
- 닫기 중심 정리 플로우
- 정리 전 미리보기와 최종 확인
- 선택한 탭 닫기
- 마지막 정리 Undo
- 최근 정리 기록 표시

## 설치 방법

1. Chrome 주소창에서 `chrome://extensions`를 연다.
2. 오른쪽 위의 Developer mode를 켠다.
3. `Load unpacked`를 누른다.
4. 프로젝트 루트 폴더를 선택한다.
5. Chrome 툴바에서 TabTidy 확장프로그램 popup을 연다.

## 사용 방법

1. popup에서 현재 창 탭 요약을 확인한다.
2. `상세 분석 열기`를 눌러 dashboard를 연다.
3. dashboard에서 탭 그룹, 비그룹 탭, 오래 안 본 탭, 중복 탭, 검색 결과 탭을 확인한다.
4. 중요한 탭은 `보호`를 눌러 닫기 후보에서 제외한다.
5. 필요한 탭에는 `메모`를 남긴다.
6. `정리하기`에서 닫을 후보를 선택한다.
7. `정리 전 미리보기`에서 닫을 탭 목록을 확인한다.
8. `정리 실행`을 누른 뒤 confirm 문구를 승인한다.
9. 필요하면 `마지막 정리 되돌리기`로 닫은 URL을 다시 연다.
10. `정리 기록`에서 최근 정리 결과를 확인한다.

## 권한 설명

- `tabs`: 현재 창 탭을 분석하고, 사용자가 선택한 탭을 닫고, Undo 시 URL을 다시 열기 위해 사용한다.
- `tabGroups`: Chrome 탭 그룹 이름, 색상, 그룹별 탭 수를 읽기 위해 사용한다.
- `storage`: 탭 사용 기록, 보호 상태, 메모, 정리 기록을 저장하기 위해 사용한다.

- 그룹 이동 기능 없음
- 자동 삭제 없음
- 자동 정리 없음
- AI 분류 없음
- 사용자 도메인 규칙 없음
- 사용자가 선택한 탭만 닫음
- 정리 실행 전 미리보기 필수
- 정리 실행 전 confirm 필수
- 정리 실행 중 버튼 중복 클릭 방지

- pinned 탭

- 사용자가 보호한 탭
- `chrome://`
- `chrome-extension://`
- `about:`
- `edge://`
- TabTidy dashboard 같은 TabTidy 내부 페이지

## 보호 탭과 메모

보호 탭은 사용자가 직접 닫기 후보에서 제외하려는 탭이다. 보호 상태는 `chrome.storage.local`의 `tabMeta.isProtected`에 저장된다.

탭 메모는 `tabMeta.note`에 저장된다. 메모가 있는 탭은 목록과 정리 후보에서 표시되며, 사용자가 닫기 전에 다시 판단할 수 있게 돕는다. 메모가 있다고 자동으로 닫기 제외되지는 않는다.

## Undo 한계

Undo는 마지막 정리에서 닫은 탭의 URL을 다시 여는 기능까지만 제공한다.

dashboard의 되돌리기 안내도 같은 내용을 표시한다. 되돌리기는 닫힌 탭 URL을 새로 여는 방식이며, 완전한 브라우저 상태 복구 기능이 아니다.

복구를 보장하지 않는 항목:

- 페이지 내부 입력 내용
- 스크롤 위치
- 영상 재생 위치
- 웹앱 내부 상태
- 브라우저 탭 히스토리
- 원래 탭 그룹

Undo는 마지막 정리 1회에 대해 한 번만 실행된다. Undo를 실행하면 `lastCleanupSnapshot`은 삭제하지 않고 `undoUsed: true`로 표시된다.

## 제한사항

보호 탭과 탭 메모는 현재 열린 탭의 `tabId` 기반으로 관리되므로, 탭을 닫았다가 다시 열거나 브라우저 세션이 바뀌면 동일 URL이라도 보호/메모가 유지되지 않을 수 있다. 이 MVP에서는 현재 열린 탭 작업공간 정리를 우선한다.

오래 안 본 탭 분석은 TabTidy 설치 이후부터 정확해진다. 설치 전이나 추적 시작 전의 탭 사용 기록은 알 수 없다.

비그룹 탭의 도메인별 묶음은 상태 파악용이다. 공부, 게임, 개발 같은 의미 분류를 하지 않는다.

## 테스트 시나리오

### 확인된 테스트 결과 체크리스트

- 보호 탭을 켜면 정리 후보에서 제외된다.
- 보호 해제하면 다시 후보로 들어온다.
- 탭 메모 추가, 수정, 삭제가 가능하다.
- dashboard 새로고침 후에도 현재 열린 탭의 메모가 유지된다.
- 중복 탭 정리 실행 후 정리 기록이 남는다.
- 되돌리기를 실행하면 닫힌 탭 URL이 다시 열린다.
- pinned 탭은 자동 제외된다.
- 현재 활성 탭은 자동 제외된다.
- `chrome-extension://` 내부 페이지는 자동 제외된다.
- TabTidy dashboard 탭은 자동 제외된다.

### 예외 상황 점검

- 탭이 1개만 있어도 popup과 dashboard가 깨지지 않는다.
- 탭 그룹이 없어도 그룹 섹션이 빈 상태로 표시된다.
- 중복 탭이 없어도 중복 섹션과 정리 후보 섹션이 빈 상태로 표시된다.
- 보호 탭만 있는 경우 닫기 후보가 비어 있고 정리 실행이 비활성화된다.
- Undo를 한 번 실행한 뒤에는 Undo 버튼이 비활성화되고, 다시 실행되지 않는다.

### 최종 테스트 시나리오

- popup에서 전체 탭 수, 그룹 탭 수, 비그룹 탭 수, 탭 그룹 수가 맞는지 확인한다.
- dashboard에서 그룹명, 그룹 색상, 그룹별 탭 수, 그룹별 건강 점수가 표시되는지 확인한다.
- 비그룹 탭이 도메인별로 묶여 보이는지 확인한다.
- 같은 URL 탭과 UTM 파라미터만 다른 URL 탭이 중복으로 감지되는지 확인한다.
- 검색 결과 탭이 검색 결과 섹션에 표시되는지 확인한다.
- 오래 안 본 탭 섹션이 설치 이후 기록 기준으로 표시되는지 확인한다.
- 일반 탭을 보호한 뒤 정리 후보에서 제외되는지 확인한다.
- 보호를 해제하면 다시 닫기 후보로 선택 가능한지 확인한다.
- 탭 메모를 추가, 수정, 삭제하고 dashboard 새로고침 후에도 현재 열린 탭에서 유지되는지 확인한다.
- 중복 탭 후보는 기본 선택되고, 오래 안 본 탭과 검색 결과 탭은 기본 미선택인지 확인한다.
- 정리 전 미리보기 없이 실행할 수 없는지 확인한다.
- confirm에서 취소하면 탭이 닫히지 않는지 확인한다.
- confirm에서 승인하면 선택한 탭만 닫히는지 확인한다.
- Undo를 누르면 닫은 탭 URL들이 다시 열리고 Undo 버튼이 비활성화되는지 확인한다.
- 정리 기록에 최신 정리 결과가 표시되는지 확인한다.
- 코드 검색으로 `chrome.tabs.group`, `chrome.tabs.move`, `chrome.tabGroups.update`가 없는지 확인한다.

## 향후 개선 계획

- 보호/메모를 tabId가 아니라 URL 또는 도메인 기반으로 선택 저장하는 옵션
- 중복 감지 고도화
- 정리 기록 내보내기
- dashboard 섹션 필터링과 검색
- 접근성 개선
- 아이콘과 배포용 패키징
